# GoHolo — Quest 3 Pinch & Distance Grab (Meta Building Blocks)

Native Meta XR Interaction SDK integration. No custom grab scripts — everything via Meta Building Blocks.

## What's in place

```

├── Robot/
```

## First-time setup

1. Open the Unity project (`ABBGoFa_Quest3`).
2. Wait for asset reimport to finish (Console clean, no compile errors).
3. Menu: **`GoHolo` → `Setup Pinch Drag Scene (overwrite)`**.
4. Confirm the overwrite dialog.
5. When the success dialog appears, press **Ctrl+B** (Build And Run).

The setup builds:
- OVRInteractionComprehensive rig (hands + all Meta interactors wired) + Hmd refs auto-fixed
- Meta Desk prefab at (0, 0, 0.8)
- Rparak GoFa ABB_CRB_15000 on desk top with all materials converted to URP Lit
- `GoFa_CRB15000.prefab` with:
  - `GoFaCCDIK` attached to root, joints array populated from Joint_1..Joint_6
  - `IKHandle` (green sphere, 6 cm) at flange with `Grabbable + GrabFreeTransformer + HandGrabInteractable + DistanceHandGrabInteractable`
  - `RotaryHandle_1..6` (white sphere 2.5 cm + white arc) at each joint with `Grabbable + OneGrabRotateTransformer + HandGrabInteractable`
- Directional Key + Fill lights, reflection probe, soft shadows

## Runtime behaviour

- **End-effector near grab**: bring your pinch (thumb+index) within a few cm of the green sphere → Meta HandGrab snaps, drag moves the handle, CCDIK follows → all 6 joints animate.
- **End-effector distance grab**: aim the hand pointer at the green sphere from further away → pinch → Meta's distance-grab ray snaps, drag moves the handle.
- **Per-joint rotation**: pinch a white handle at any joint → Meta rotates the parent joint around its local Z axis, clamped to URDF limits (arc shows the allowed range).

## Editing the generated scene

- Treat `PinchDragGoFa.unity` like any Unity scene — Ctrl+S saves, edits persist.
- **Never rerun** `Setup Pinch Drag Scene (overwrite)` unless you want to reset manual edits.
- The generated robot is a **Prefab** at `Assets/GoHolo/Prefabs/GoFa_CRB15000.prefab`; open it in Prefab editor to inspect the Meta-component wiring.

## Known limits / next steps

- **URP shading**: `ConvertMaterialsToURP` maps main texture + color + metallic/smoothness only. Detail textures from Meta's custom `Oculus/Interaction/Room` shader are ignored (the shader is built-in pipeline-only).
- **Arc visual**: `JointArcVisual` is a minimal LineRenderer arc. Meta's official `ArcAffordanceController` with animated skinned mesh is not used here — would require extracting the rig from `PanelWithManipulators.prefab`.
- **EGM streaming**: `egm.proto` is in `Robot/PROTO/` but no client code yet. Next phase: regenerate C# bindings via modern `protoc` + `Google.Protobuf` (not the legacy 2.4 from rparak), plumb joint angles from `GoFaCCDIK.GetJointAnglesDeg()` into UDP packets on port 6511.
- **Safety**: nothing enforced yet (no workspace limits, no collision avoidance). See `C:\GoHolo\HANDOFF.md` phase 10.
